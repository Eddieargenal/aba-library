## Work Units

### WU-1: Create `scripts/build-index.py` and run it
- **Objective:** Write a Python 3 script (standard library only) that parses frontmatter from all 5 page-type directories and emits `indexes/agent-index.md`. Then run it to generate the index.
- **Inputs:**
  - Architecture doc sections: "Navigation and Index Generation", "Frontmatter Schema"
  - Page-type directory table above
  - `indexes/` directory (output destination)
- **Output:**
  - `scripts/build-index.py` — executable script
  - `indexes/agent-index.md` — generated index, organized by type, then tier (for frameworks). Each entry: file path, title, lifecycle_stage, status, type-specific metadata (source_count for concepts, tier for frameworks, field_instruments count for tools).
- **Dependencies:** None (can run in parallel with WU-3 and WU-4)
- **Complexity tier:** Tier 2
- **Validation method:**
  - Script runs without error: `python3 scripts/build-index.py`
  - Output file exists: `indexes/agent-index.md`
  - Line count check: index must contain entries for all 123 pages (minus 00_index.md files)
  - Check that malformed YAML files are skipped gracefully, not crashing the script

**Script spec:**
```
VAULT = "/Users/eddieargenal/Documents/obsidian-vault"
DIRS = {
    "source":           "wiki/aba/01-sources/extracted",
    "concept":          "wiki/aba/02-concepts",
    "framework":        "wiki/aba/03-frameworks",
    "tool":             "wiki/aba/04-tools",
    "field-instrument": "wiki/aba/05-field-instruments",
}
EXCLUDE = {"00_index.md"}

For each page type, for each .md file (excluding EXCLUDE):
  - parse frontmatter (try/except — skip on error, print warning to stderr)
  - extract: title, status, lifecycle_stage, type-specific fields
  - add to type-section of output

Output format:
  indexes/agent-index.md — markdown file with:
  - header: generated timestamp, total count
  - one H2 section per page type
  - within frameworks: H3 subsections for Tier 1 and Tier 2
  - markdown table per section: | path | title | status | lifecycle_stage | [type-specific] |
```
