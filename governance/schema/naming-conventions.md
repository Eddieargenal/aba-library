# Naming Conventions

## Raw source files
Convention: YYYY-org-or-author-short-title-source-type-version.ext
Rules: lowercase kebab-case; identify the SOURCE DOCUMENT, not derived tools; preserve original filename in metadata.

Examples:
- 2017_sanderson-sitko_urban-area-based-approaches-post-disaster-guide.pdf
- 2019-replication-scale-up-learning-note-framework.pdf
- 2015_undrr_sendai-framework-drr-2015-2030-framework.pdf

## Wiki pages
Convention: kebab-case description of the concept, tool, or instrument.
Tool pages: prefix with two-digit number (01-, 02-, ...).
Lifecycle pages: prefix with two-digit number (00-, 01-, ...).

## IDs
source_id: same as canonical filename (without extension)
tool_id: tool-NN-short-name
instrument_id: instrument-short-name
