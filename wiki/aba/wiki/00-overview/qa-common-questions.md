# Quick Answers — Common Field Questions

> Read this before hunting through tool pages. If your question is here, you have your answer in one read.

## Q1: What exactly is an area-based approach?

**Three conditions, all required:**
1. **Geographically targeted** — selected based on need and vulnerability concentration, not administrative convenience
2. **Multi-sector** — addresses multiple interdependent sector needs in an integrated way
3. **Participatory** — meaningful community involvement in assessment, design, and accountability

**Not ABA:** A program is NOT area-based simply because it is located in a neighborhood.

→ Misuse patterns: [[./12risks-contradictions/common-aba-misuse]]

## Q2: "We work in one neighborhood, so we're area-based." Is that right?

**No.** Geographic placement ≠ area-based approach.

The ABA label requires all three criteria above. A single-sector program in a neighborhood is a geographically located sector program. It is not an ABA unless it also coordinates across sectors and involves the community in design decisions.

**How to push back:**
- Ask: Are needs addressed across 3+ interdependent sectors? If no → not ABA.
- Ask: Is there a functioning community engagement mechanism shaping program design? If no → not ABA.
- Ask: Is there a coordination structure preventing duplication with other actors? If no → not ABA.

→ Source: Parker & Maynard (2015): [[./01sources/2015-parker-maynard-humanitarian-response-urban-crises-aba-review]]
→ Misuse patterns: [[./12risks-contradictions/common-aba-misuse]]
→ One-pager: [[./02concepts/aba-definition-one-pager]]

## Q3: Should we use an ABA in this context?

**Use Tool #01.** It scores 8 domains and produces a decision category.

**Quick version — ask these 5 questions:**
1. Are needs geographically concentrated (>40% of most-affected population in one area)?
2. Are 3+ sectors failing in interdependent ways?
3. Are urban systems (water, markets, transport, services) driving the problem?
4. Are 4+ actors present with uncoordinated activities?
5. Is meaningful community participation feasible in this timeline?

If ≥3 yes → run the full tool.
If ≤1 yes → consider sector-only or household approach.

## Q5: How do we calculate the ABA score?

**Score = sum of (raw score / 4 × weight × 100) across 8 domains.**

| Domain | Weight |
|---|---:|
| Geographic concentration | 15% |
| Multi-sector need | 15% |
| Urban systems interdependence | 15% |
| Stakeholder/coordination complexity | 15% |
| Local capacity/resource leverage | 10% |
| DRR/future risk relevance | 15% |
| Participation feasibility | 10% |
| Operational feasibility/safeguards | 5% |

| Score | Decision |
|---:|---|
| 80–100 | Full ABA justified |
| 65–79 | ABA justified with conditions |
| 50–64 | Hybrid model |
| 35–49 | Sector/household approach preferred |
| 0–34 | Do not use ABA |

→ Full scoring worksheet: [[./04tools/01-aba-feasibility-and-necessity-assessment-tool]]

## Q6: How does the Sendai Framework connect to ABA?

**Four Sendai priorities:**
1. Understanding disaster risk (→ HEVC mapping, community risk assessment)
2. Strengthening disaster risk governance (→ municipal alignment, local DRR plans)
3. Investing in resilience (→ BBB in shelter/infrastructure design, community preparedness)
4. Enhancing preparedness for effective response (→ early warning, contingency planning)

**ABA connection:** An ABA program contributes to Sendai when it (a) collects and uses risk data in area selection, (b) aligns with municipal DRR plans, (c) designs for risk reduction not just need response, and (d) strengthens local preparedness capacity.

→ Source: [[./01sources/2015-undrr-sendai-framework-drr-2015-2030-framework]]
→ Tool: Module 7 (DRR Relevance) in [[./04tools/01-aba-feasibility-and-necessity-assessment-tool]]

## Q7: Two NGOs are doing shelter in our target area. What do we do?

1. Collect 5W data for both actors: [[./05field-instruments/stakeholder-5w-mapping-form]]
2. Map overlaps: same activity + same area + same population = duplication
3. Check if it's true duplication (same households) or complementary (adjacent blocks, different groups)
4. If duplication: raise in coordination meeting using [[./08coordination/coordination-meeting-brief-template]]
5. Log the decision: [[./08coordination/decision-log-template]]
6. Adjust your targeting or activity scope to eliminate overlap

→ Protocol: [[./08coordination/duplication-detection]]
→ Instrument: [[./05field-instruments/duplication-gap-analysis-matrix]]

## Q8: What are the known risks of using an ABA?

**Top 5 flags (check before recommending ABA):**
1. Area targeting can expose beneficiaries to stigma or community tension
2. Community consultation can reproduce elite capture if not inclusive
3. Municipal alignment can co-opt programs in politicized contexts
4. ABA creates coordination overhead — may not be worth it for small, single-actor programs
5. Participation requirements may be impossible in acute emergency phases

→ Full list: [[./12risks-contradictions/protection-and-do-no-harm-risks]]

## Q9: What must be true before handing over the program to local actors?

**Minimum handover conditions:**
- Handover partner identified and has confirmed willingness and mandate
- Handover partner has minimum operational capacity (staff, budget, systems)
- Community is informed and accepts the transition
- All accountability obligations (complaints, unresolved cases) are resolved or transferred
- Services are at a sustainable level without external subsidy
- DRR/BBB elements are embedded in local systems, not dependent on the program
