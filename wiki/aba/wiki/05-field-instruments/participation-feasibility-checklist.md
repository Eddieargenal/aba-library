# Participation Feasibility Checklist

## Purpose
A structured checklist for assessing whether meaningful community participation is feasible in the target area — considering safety, inclusion, time, language, accessibility, and institutional mechanisms. Feeds directly into Domain 8 scoring in Tool #01.

## When to use
Stage 0 (appropriateness decision). Must be completed before finalizing ABA decision.

## Who collects / facilitates
Protection-trained field staff in consultation with community leaders and protection actors.

## Respondents / participants
Field team assessment drawing on KIIs with community leaders, protection actors, and own observation.

## Participation Feasibility Checklist

For each criterion: Rate as YES / PARTIAL / NO. Add evidence note. Flag PARTIAL or NO responses.

### A: Safety and security
| Criterion | Rating | Evidence | Risk flag |
|---|---|---|---|
| Community members can attend meetings without risk to their safety | | | |
| Women can participate without safety risk or social sanction | | | |
| IDPs / displaced persons can participate without risk of exploitation or targeting | | | |
| Marginalized groups (ethnic minorities, LGBTQ+, persons with disability) can participate without risk | | | |
| No armed actors are using community spaces for recruitment or pressure | | | |

### B: Inclusion
| Criterion | Rating | Evidence | Risk flag |
|---|---|---|---|
| Women can be reached through separate or safe consultation processes | | | |
| Youth (15-24) can participate in dedicated mechanisms | | | |
| Persons with disability can access venues and processes | | | |
| Non-dominant language speakers can participate (translation available) | | | |
| IDPs and host community can participate without domination by one group | | | |
| Gatekeeping by powerful local actors can be mitigated | | | |

### C: Mechanisms and capacity
| Criterion | Rating | Evidence | Risk flag |
|---|---|---|---|
| A viable community meeting venue exists that is accessible and neutral | | | |
| Community leaders or CBOs exist who can organize consultations | | | |
| Feedback mechanisms (hotlines, complaint boxes, community monitors) can be set up | | | |
| Timelines allow for minimum 2 rounds of community consultation before area strategy is finalized | | | |
| Staff capacity exists to facilitate meaningful consultations (language, facilitation skills) | | | |

### D: Context-specific constraints
| Criterion | Rating | Evidence | Risk flag |
|---|---|---|---|
| No active conflict or displacement event is making consultation currently impossible | | | |
| Access is sufficient to reach participation venues for all target groups | | | |
| Cultural or religious constraints on participation have been identified and mitigated | | | |
| No coercive actor (government, militia, powerful CBO) will manipulate consultation outcomes | | | |

## Summary assessment
- Total YES: ___ / Total PARTIAL: ___ / Total NO: ___
- Number of items rated NO: ___
- Participation feasibility level:
  - [ ] HIGH — 18+ YES, no NO items: full participation feasible
  - [ ] MEDIUM — 12-17 YES, 1-3 PARTIAL: participation feasible with targeted mitigations
  - [ ] LOW — <12 YES or any NO: participation design requires significant safeguards; document risks
  - [ ] INFEASIBLE — 5+ NO items or key safety criteria rated NO: do not proceed with ABA unless resolved

## Enumerator / facilitator guidance
Do not score this checklist based only on general optimism about participation. Be concrete about what evidence exists for each criterion. A YES requires specific evidence, not assumption.

## Data quality checks
- Must include input from at least 1 protection actor (see kii-guide-protection-actors)
- Any NO on safety criteria is a potential red flag for Tool #01
- Do not override a NO rating without concrete evidence of changed circumstances

## Analysis use
Score Domain 8 (participation feasibility) in Tool #01 directly from this checklist. Any NO items should be flagged as potential red flags.

## Risks / ethical safeguards
- Participation that appears inclusive on paper may be tokenistic in practice
- Check back after first community consultations whether participation quality matches design
- Document participation barriers honestly — for accountability and adaptive management
