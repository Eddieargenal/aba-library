# 03 — Frameworks

9 analytical frameworks structuring decision-making across the ABA/DRR programme lifecycle. Only the ABA appropriateness decision framework is fully populated; the other 8 are stubs. Use when a task requires a structured approach to a specific decision.

## Documents

| Document | Description |
|---|---|
| [[aba-appropriateness-decision-framework]] | **What:** Evidence-based framework defining 9 decision domains, a weighted scoring model (0–100), red-flag overrides, and minimum evidence standards for determining whether ABA is justified. **Why:** The conceptual foundation for Tool #01 — explains the logic and evidence requirements behind the scoring, not just the score itself. **When/How:** Read when you need to explain the reasoning behind an ABA appropriateness decision or when Tool #01 scoring alone is insufficient justification for partners. |
| [[area-based-coordination-framework]] | **What:** Framework defining the architecture, roles, responsibilities, and operating procedures for an area-based coordination platform and its relationship with sector/cluster coordination and municipal authorities. **Why:** Ensures multi-agency coordination in ABA programming is structured, avoids duplication, and aligns with municipal systems and existing cluster architecture. **When/How:** Read when designing or assessing the coordination platform for an ABA programme, or when using Tool #06 (Stakeholder Mapping) or Tool #13 (Coordination Dashboard) to manage coordination workflows. |
| [[area-selection-framework]] | **What:** Framework for selecting and prioritizing urban neighborhoods for area-based programming using criteria that combine humanitarian severity, DRR risk, operational feasibility, and strategic value. **Why:** Provides transparent, defensible selection logic so geographic targeting decisions can be justified to partners, donors, and affected communities. **When/How:** Read when determining which areas to target for ABA/DRR programming or when applying Tool #02 (Area Selection Matrix) and Tool #03 (Boundary Definition). |
| [[implementation-adaptation-framework]] | **What:** Framework guiding structured implementation of area-based programs through sequencing logic, adaptive management triggers, coordination mechanisms, and real-time learning processes. **Why:** Enables programs to adjust to changing conditions on the ground while maintaining accountability, rather than rigidly following pre-set plans. **When/How:** Read when planning or adapting implementation activities during area-based programming, or when using Tool #12 (Sequencing & Dependency Map) or Tool #16 (M&E Adaptation Framework). |
| [[integrated-area-strategy-framework]] | **What:** Framework for developing a coherent area-level response strategy integrating multi-sector program design, coordination architecture, local resource leverage, and DRR objectives into a unified theory of change. **Why:** Prevents siloed sector responses by forcing strategic integration across all program elements within a defined geographic area. **When/How:** Read when synthesizing diagnosis findings into an actionable area strategy or when using Tool #10 (Strategy Builder) and Tool #09 (Response Option Comparison). |
| [[joint-prioritization-framework]] | **What:** Framework structuring the joint analysis and cross-sector prioritization of risks and needs, integrating severity, vulnerability, feasibility, DRR relevance, and protection considerations. **Why:** Ensures prioritization decisions reflect multi-stakeholder consensus and balance competing sectoral demands within a common analytical structure. **When/How:** Read when conducting joint risk and needs prioritization with partners or when populating Tool #08 (Joint Risk and Needs Prioritization Matrix). |
| [[neighborhood-diagnosis-framework]] | **What:** Framework providing a structured approach to profiling an urban area across systems status, HEVC analysis, service functionality, stakeholder landscape, and community dynamics. **Why:** Produces a comprehensive baseline understanding of the area that informs all subsequent planning, targeting, and strategy decisions. **When/How:** Read when beginning work in a new area and needing to understand the urban system, or when using Tool #04 (Systems Diagnosis), Tool #05 (HEVC Mapping), or Tool #06 (Coordination Mapping). |
| [[transition-handover-framework]] | **What:** Framework structuring the transition from humanitarian ABA programming to local or municipal ownership with defined handover conditions, actors, resources, and unresolved risk management. **Why:** Prevents gains from being lost when humanitarian actors exit by systematically transferring capacity, responsibility, and resources to sustainable local structures. **When/How:** Read when planning the exit or scale-up of an area-based programme or when using Tool #17 (Handover & Scale-Up Checklist). |
| [[urban-drr-response-design-framework]] | **What:** Framework structuring the design of DRR and emergency response programs in urban settings by integrating hazard analysis, systems thinking, and area-based coordination into a coherent response architecture. **Why:** Grounds response design in Sendai Framework principles while adapting them to the complexity and density of urban disaster contexts. **When/How:** Read when translating urban risk assessments into actionable DRR response designs or when using Tool #04 (Systems Diagnosis), Tool #05 (HEVC Mapping), or Tool #10 (Strategy Builder) to inform response architecture. |
