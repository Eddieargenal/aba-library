# 00 — Overview

Navigation, operating rules, and quick-reference orientation for agents and users entering the wiki. Contains the canonical agent contract, operating model, common Q&A, and knowledge map. Read before answering domain questions or running any wiki operation.

## Documents

| Document | Description |
|---|---|
| [[agent-contract]] | **What:** The canonical operating contract defining rule precedence and layer discipline for all agents. **Why:** Ensures consistent agent behaviour — this file overrides all other guidance when instructions conflict. **When/How:** Read first when any instruction is ambiguous; this file is the single source of truth for operating rules. |
| [[agent-operating-model]] | **What:** Describes the four agent task types — answering questions, ingesting sources, generating tools, and generating field instruments. **Why:** Orients agents to the scope of operations this wiki supports. **When/How:** Read when setting up agent workflows or when unsure what operations are permitted. |
| [[how-to-use-this-wiki]] | **What:** Step-by-step guidance for AI agents and human users on how to navigate and use the wiki correctly. **Why:** Reduces navigation errors and ensures agents answer from the wiki layer, not raw sources. **How:** Follow the numbered steps for agents; human users start from the knowledge map. |
| [[qa-common-questions]] | **What:** Pre-answered common field questions covering ABA definition, appropriateness, area selection, sector application, and coordination. **Why:** The fastest path to answers — one read gives the answer without navigating tool or concept pages. **When:** Read first before opening any tool page; if your question is here, it is already answered. |
| [[source-catalog-from-matrix]] | **What:** Full 46-row source catalog parsed from the Urban ABA DRR matrix spreadsheet, mapping lifecycle stages to tools, sources, PDFs, and ingestion status. **Why:** The authoritative cross-reference between programme stages and their primary sources. **How:** Use when identifying which sources are relevant to a specific lifecycle stage or tool. |
| [[tool-01-quick-reference]] | **What:** Compact scoring model and core decision rules for Tool 01 (ABA Feasibility and Necessity Assessment). **Why:** Provides fast access to decision thresholds and module structure without loading the full 9-module tool page. **When:** Use for quick scoring checks; link to the full tool page for evidence collection procedures. |
| [[urban-drr-aba-knowledge-map]] | **What:** Architecture overview of the wiki — purpose, knowledge layers, and which sections cover which topics. **Why:** Gives agents and users a mental model of the wiki before navigating it. **How:** Read when starting a new session or when unsure which section to consult for a given question type. |
| [[library-architecture]] | **What:** Mermaid diagram of the full library architecture — all sections, their roles, relationships, and the agent navigation flow. **Why:** Visual reference for understanding how the library is structured at a glance. **When:** Use when orienting a new agent or user, or when redesigning the library structure. |
