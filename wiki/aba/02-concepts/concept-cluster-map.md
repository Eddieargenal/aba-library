# Concept Cluster Map — Phase 1

This document records the output of Phase 1 of the bottom-up concept identification process. All concept families were derived from the `# Key concepts` sections across 23 files in `01-sources/extracted/`. No concepts were added from general knowledge.

**Process:** Raw candidates were extracted from all 23 extracted source files, then clustered into families by shared referent. Source count, definition convergence, and operationalizability were scored per family. Build priority tiers reflect source count and strategic importance to the wiki's ABA/DRR focus.

**Next step:** Phase 2 — full-file pass per concept family across all sections of all extracted files, producing a concept page per family.

---

## Scoring key

| Field | Values |
|---|---|
| **Convergence** | Consistent = sources agree / Partial = core shared, variants differ / Contested = sources genuinely conflict |
| **Operationalizable** | Yes = source attaches a tool or method / Partial = implicit instrument / No = purely conceptual |
| **Tier** | 1 = build first / 2 = build second / 3 = build last |

## Structural observations

**1. The ABA definition is the most important and most contested.**
Eight sources define it, but they do not fully agree. Parker & Maynard's (2015) requirement that all three criteria be present simultaneously is the strictest and most cited formulation, but several sources treat geographic focus, multi-sectorality, and participation as tendencies or design variables rather than hard requirements. The concept page must name this tension rather than resolve it artificially.

**2. Urban resilience requires splitting or explicit bracketing.**
Four definitions from four sources are not interchangeable and serve different analytical purposes (community capacity, systems efficiency, structural preservation, long-term trajectory). A single page presenting all four with clear attribution is more honest and more useful than a synthetic definition.

**3. The neighbourhood concept is the most contested territory.**
Three sources dedicated to defining "neighbourhood" produce three different typologies, and none resolves the others. The concept page's job is to present the debate, not settle it.

**4. One candidate merger.**
"Geographic targeting," "boundary definition," and "scale as design variable" are sufficiently related that they could share one concept page — or remain separate if the wiki needs distinct entry points for area selection logic versus boundary legitimacy versus scale choice. This decision should be made before Phase 2 begins.

## Build sequence (Phase 2)

Build concept pages in Tier 1 → Tier 2 → Tier 3 order.

Within Tier 1, build ABA first — it is the wiki's organizing concept and most other Tier 1 concepts are partially defined in relation to it.

Within Tier 2, build Urban Resilience before Neighbourhood Concept — it has more source grounding and its contested definitions are more directly actionable.

Do not build Tier 3 concepts until Tier 1 and Tier 2 are complete, except where a Tier 3 concept is needed as a link target from a Tier 1 or Tier 2 page.
