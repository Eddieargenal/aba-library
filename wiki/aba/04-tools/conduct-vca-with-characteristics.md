<a id="purpose"></a>
## Purpose

Structure the **analysis** of participatory VCA data using the Characteristics — without pre-defining indicators — so field workers know the **range of vulnerabilities to look for** while keeping the assessment open.

<a id="decision-questions"></a>
## Decision Questions

- *"Our VCA produces too much messy data — how do I structure analysis without biasing it?"*
- Which thematic areas are under-represented in what the community raised?

<a id="method"></a>
## Method

Run the VCA participatorily and openly; **after** collection, use the Characteristics as a lens to organise findings and spot gaps. Do **not** convert the Characteristics into a pre-set questionnaire (that biases collection).

<a id="risks"></a>
## Risks

Letting the framework pre-structure data collection (defeats participation); over-fitting messy field data to tidy categories.
