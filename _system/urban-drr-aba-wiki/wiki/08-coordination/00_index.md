# 08 — Coordination

Structures, tools, and protocols for multi-actor area-based coordination and information management. Use when designing coordination architecture, managing 4W/5W data, or establishing inter-agency referral and decision-making protocols. Three template files are fully populated; the rest are stubs awaiting extraction from source documents.

## Documents

| Document | Description |
|---|---|
| [[actor-responsibility-matrix]] | **What:** A matrix template clarifying which actor leads, supports, monitors, and coordinates each activity domain in the area. **Why:** Prevents accountability gaps when multiple organizations operate in the same location. **When/How:** Complete after initial 5W mapping; review at each coordination cycle; flag domains with no lead actor assigned. |
| [[coordination-meeting-brief-template]] | **What:** A standardized template for area coordination meeting briefs covering agenda, attendance, decisions, action points, and gap/duplication reporting. **Why:** Ensures institutional memory and accountability across actor turnover. **When/How:** Use for every area coordination meeting; number meetings sequentially and archive all briefs. |
| [[decision-log-template]] | **What:** A template for logging coordination decisions with rationale, responsible actor, deadline, and status. **Why:** Provides accountability and memory for decisions made across coordination meetings. **When/How:** Number decisions sequentially within the area; review open decisions at every meeting; flag items overdue by more than 2 weeks. |
