<a id="purpose"></a>
## Purpose

Provide a **milestones model** that measures a community's progress from its current state toward the ideal resilience state, so improvement can be shown incrementally rather than as a binary resilient/not-resilient judgement.

<a id="decision-questions"></a>
## Decision Questions

- "How do we show **gradual improvement** in resilience, not just yes/no?" (source field query)
- Where on the path from current to ideal does each characteristic sit today?
- Which milestones are realistic within the project horizon?

<a id="method"></a>
## Method

Define, per characteristic (or thematic area), an ordered set of milestones between baseline and the ideal state; assess current position; track movement across review cycles. Seeded from the source's milestones concept (p. 19); a validated scoring/weighting scheme is **not yet defined** (noted gap).

<a id="risks"></a>
## Risks

Milestone sets that encode the assessor's assumptions; false precision from unvalidated scoring. Use alongside participatory assessment, not as a standalone scorecard.
